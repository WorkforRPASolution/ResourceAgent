========================================================
  ResourceAgent 설치 가이드
========================================================

[사전 준비]

  1. 이 폴더를 PC의 적절한 위치에 압축 해제합니다.
     예: C:\Temp\install_package_windows\

  2. 관리자 권한이 필요합니다.


[설치 방법]

  1. install_ResourceAgent.bat 파일을 우클릭 → "관리자 권한으로 실행"

     * ARSAgent가 이미 설치되어 있으면 설치 경로를 자동으로 감지합니다.
     * ARSAgent가 없으면 /basepath 옵션을 지정해야 합니다.
     * LhmHelper + PawnIO 드라이버가 기본 포함됩니다.
     * LhmHelper는 .NET Framework 4.8 이상이 필요합니다.
       - 설치되어 있지 않으면 오프라인 설치기가 자동 실행됩니다 (~112MB).
       - 설치 후 재부팅이 필요할 수 있으며, 이 경우 서비스는 등록만 되고
         재부팅 이후 자동 시작됩니다 (start=auto).
       - Windows Update로 4.8 이상이 설치된 PC는 자동 스킵됩니다.

  2. 하드웨어 센서 수집이 필요 없는 경우:
     명령 프롬프트를 관리자 권한으로 열고 다음을 실행합니다.

       install_ResourceAgent.bat /nolhm

     이 옵션은 LhmHelper와 PawnIO 드라이버 설치를 건너뜁니다.

  3. 설치 경로를 수동으로 지정하려면:

       install_ResourceAgent.bat /basepath E:\MyPath\EEGAgent
       install_ResourceAgent.bat /basepath E:\MyPath\EEGAgent /nolhm

  4. VirtualAddressList를 직접 지정하려면:

       install_ResourceAgent.bat /site 10.0.0.1,10.0.0.2

  5. 파일이 이미 복사되어 있는 경우 (원격 배포 등):
     서비스 등록 + PawnIO 드라이버 설치만 수행합니다.

       install_ResourceAgent.bat /basepath E:\MyPath\EEGAgent /nocopy /site 10.0.0.1


[설치 확인]

  설치가 완료되면 다음 메시지가 표시됩니다:

    ResourceAgent installed and running successfully!
      BasePath: D:\EARS\EEGAgent
      Binary:   D:\EARS\EEGAgent\bin\x86\ResourceAgent.exe
      Config:   D:\EARS\EEGAgent\conf\ResourceAgent\
      Logs:     D:\EARS\EEGAgent\log\ResourceAgent\

  또는 명령 프롬프트에서 확인:

    sc.exe query ResourceAgent


[제거 방법]

  명령 프롬프트를 관리자 권한으로 열고:

    install_ResourceAgent.bat /uninstall

  설치 경로는 등록된 서비스 정보에서 자동으로 감지됩니다.
  (/basepath 옵션을 별도로 지정할 필요 없음)

  서비스 중지 → PawnIO 드라이버 제거 → 파일 삭제 순으로 진행됩니다.
  (ARSAgent 파일은 삭제되지 않습니다)


[Windows 32-bit 설치]

  Windows 7 32-bit 전용 패키지를 사용합니다.
  (패키지명: install_package_windows_x86.zip)

  설치 방법은 64-bit와 동일합니다. install_ResourceAgent.bat 실행.

  차이점:
    - 32-bit ResourceAgent.exe가 bin\x86\ 에 포함됩니다.
    - LhmHelper/PawnIO는 64-bit 전용이므로 패키지에 미포함됩니다.
      Windows 7 32-bit에서는 하드웨어 센서(온도, GPU 등) 수집이 불가하며,
      CPU/Memory/Disk/Network 등 OS 메트릭은 정상 수집됩니다.


[Linux 설치]

  sudo ./install_ResourceAgent.sh --base-path /opt/EEGAgent

  옵션:
    --base-path PATH    설치 경로 (ARSAgent 자동 감지)
    --user USER         서비스 실행 사용자 (기본: resourceagent)
    --site ADDR         VirtualAddressList 직접 지정
    --nocopy            파일 복사 생략 (서비스 등록만 수행)
    --uninstall         제거

  예시:
    sudo ./install_ResourceAgent.sh                                          # ARSAgent 경로 자동 감지
    sudo ./install_ResourceAgent.sh --base-path /opt/EEGAgent --site 10.0.0.1
    sudo ./install_ResourceAgent.sh --base-path /opt/EEGAgent --nocopy --site 10.0.0.1
    sudo ./install_ResourceAgent.sh --uninstall


[설치 후 폴더 구조]

  D:\EARS\EEGAgent\
  ├── bin\x86\
  │   └── ResourceAgent.exe        ← 모니터링 에이전트
  ├── conf\ResourceAgent\
  │   ├── ResourceAgent.json       ← 메인 설정
  │   ├── Monitor.json             ← 수집 항목 설정 (변경 시 자동 반영)
  │   └── Logging.json             ← 로그 설정 (변경 시 자동 반영)
  ├── log\ResourceAgent\
  │   ├── ResourceAgent.log        ← 에이전트 로그
  │   └── metrics.jsonl            ← 수집 메트릭 (file sender 사용 시)
  └── utils\lhm-helper\            ← 기본 포함 (/nolhm 시 제외)
      ├── LhmHelper.exe            ← C# 하드웨어 센서 헬퍼
      ├── LhmHelper.exe.config
      ├── LibreHardwareMonitorLib.dll
      ├── HidSharp.dll
      ├── System.Management.dll
      ├── System.Text.Json.dll 및 의존 DLL 다수
      └── PawnIO_setup.exe         ← Win8+ 드라이버 (Win7은 자동 스킵)


[문제 해결]

  "This script requires Administrator privileges" 오류:
    → install_ResourceAgent.bat를 우클릭 → "관리자 권한으로 실행"

  "ResourceAgent.exe not found in package" 오류:
    → 패키지가 올바르게 압축 해제되었는지 확인하세요.
       bin\x86\ResourceAgent.exe 파일이 있어야 합니다.

  "PawnIO driver installation failed" 오류:
    → 다른 하드웨어 모니터링 프로그램이 실행 중이면 종료 후 재시도

  Windows 7에서 "Windows 7 detected: skipping PawnIO driver" 메시지:
    → 정상 동작입니다. PawnIO는 Windows 8+ 전용이며,
      LhmHelper가 내장 WinRing0 드라이버로 자동 폴백하여
      온도/GPU 등 하드웨어 센서를 정상 수집합니다.

  ".NET Framework 4.8 installation failed" 오류:
    → Windows Update가 실행 중이면 종료 후 재시도
    → WSUS 정책이 .NET 설치를 차단하는 환경이면 관리자에게 문의
    → /nolhm 옵션으로 LhmHelper만 제외하여 우선 설치 가능

  "REBOOT REQUIRED before service starts" 메시지:
    → .NET Framework 4.8 설치 후 재부팅 필요 상태.
      PC를 재부팅하면 ResourceAgent 서비스가 자동으로 시작됩니다.
      (start=auto로 등록되어 있음)

  서비스가 시작되지 않을 때:
    → 로그 확인: D:\EARS\EEGAgent\log\ResourceAgent\ResourceAgent.log
    → 설정 파일 확인: conf\ResourceAgent\ResourceAgent.json


[참고]

  * 설정 파일(conf\ResourceAgent\*.json)은 대상 경로에 이미
    존재하면 덮어쓰지 않습니다. 바이너리만 업데이트됩니다.

  * Monitor.json, Logging.json은 서비스 재시작 없이
    파일 수정만으로 자동 반영됩니다 (Hot Reload).

  * ResourceAgent.json 변경은 서비스 재시작이 필요합니다:
      sc.exe stop ResourceAgent
      sc.exe start ResourceAgent
