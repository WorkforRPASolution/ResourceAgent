현장 관리자용 안내

모니터링 프로그램의 CPU 사용률이
작업 관리자와 다르게 보이는 이유

자원 수집 프로그램(ResourceAgent)이 보고하는 CPU 값과 Windows 작업 관리자 그래프가 차이 나는 현상에 대한 설명입니다.

한 줄 요약 — 모니터링 프로그램과 작업 관리자의 CPU 값이 조금 다른 것은 오류가 아니라 정상입니다. 두 도구가 CPU를 서로 다른 방식(다른 자)으로 측정하기 때문이며, 상황에 따라 어느 쪽이 더 높을지 방향도 바뀔 수 있습니다.

🔎 어떤 현상인가요?

같은 PC, 같은 시점인데도 모니터링 프로그램이 보고하는 CPU 사용률과 작업 관리자 성능 탭의 그래프가 서로 다르게 나옵니다. 예를 들어 한쪽이 40%일 때 다른 쪽은 30% 안팎으로 나올 수 있습니다.

💡 왜 차이가 생기나요?

같은 길이를 줄자로 재는 사람과 30cm 자로 재는 사람의 숫자가 조금 다를 수 있듯이, 두 도구는 Windows가 제공하는 서로 다른 "측정 자"를 사용합니다.

모니터링 프로그램

"CPU가 실제로 일한 시간"

CPU가 순수하게 작업에 매달린 시간의 비율을 잽니다. 업계 표준 방식으로, 다른 전문 모니터링 도구들도 동일하게 사용합니다.

작업 관리자 (Win 8 이상)

"CPU 속도까지 반영한 활용률"

CPU가 그 순간 얼마나 빠른 속도로 돌았는지까지 함께 계산합니다. 그래서 CPU 속도 상태에 따라 값이 달라집니다.

↕️ 어느 쪽이 더 높은지는 "그때 CPU 속도"에 달렸습니다

CPU는 항상 같은 속도로 돌지 않습니다. 한가하면 전기를 아끼려 속도를 낮추고, 바쁘면 속도를 높입니다. 작업 관리자는 이 속도까지 반영하기 때문에, CPU가 기본 속도보다 빠른지 느린지에 따라 방향이 바뀝니다.

그때 CPU 상태CPU 속도어느 값이 더 높나
한가함기본보다 느림 (절전)모니터링 값이 조금 높음
보통기본 속도 근처거의 같음
많이 바쁨기본보다 빠름 (터보)작업 관리자가 조금 높음
그래서  모니터링 값이 작업 관리자보다 조금 더 높게 나오는 것도 (CPU가 한가해서 속도를 낮춘 상태라면) 정상입니다. 둘 다 맞는 값입니다.

🪟 Windows 7에서는 두 값이 거의 같습니다

위의 "속도까지 반영" 방식은 Windows 8 이상의 작업 관리자에만 있습니다. Windows 7의 작업 관리자는 모니터링 프로그램과 똑같이 "일한 시간"만 재기 때문에, 두 값이 원래 거의 일치합니다. (차이는 측정 시점이 미세하게 다른 정도뿐)

✅ 괜찮은가요?

  • 두 값이 다른 것은 설계상 정상입니다. 어느 쪽도 틀린 값이 아닙니다.
  • CPU 상태에 따라 모니터링 값이 더 낮을 수도, 더 높을 수도 있습니다.
  • 모니터링 값은 Prometheus 등 전문 모니터링 도구와 같은 방식으로 측정됩니다.
  • 추세(올라가는지/내려가는지)를 보는 데는 두 값 모두 문제없이 쓸 수 있습니다.
이럴 때는 점검 대상  아래의 경우는 정상 범위를 벗어난 신호일 수 있으니 담당 엔지니어에게 알려 주세요.
 • Windows 7인데 두 값이 크게 벌어질 때
 • CPU가 바쁜 상태인데도 모니터링 값이 작업 관리자보다 계속 크게 높게 나올 때

❓ 자주 묻는 질문

어느 숫자를 믿어야 하나요?

둘 다 맞습니다. 모니터링 시스템에서 비교·추세를 볼 때는 모니터링 값을, 그 PC 앞에서 순간 상태를 볼 때는 작업 관리자 값을 참고하시면 됩니다.

모니터링 값이 작업 관리자보다 더 높게 나오는데 문제인가요?

대개는 정상입니다. CPU가 한가해서 속도를 낮춘 상태라면 모니터링 값이 오히려 조금 높게 나올 수 있습니다. 다만 바쁜 상태인데도 계속 크게 높다면 담당 엔지니어에게 문의하세요.

차이가 어느 정도까지 정상인가요?

보통 10% 안팎이면 정상 범위이고, CPU가 한가할 때는 차이가 거의 없습니다. 비정상적으로 큰 차이가 지속되면 알려 주세요.

설정을 바꿔야 하나요?

아니요. 측정 방식의 차이이므로 별도 조치가 필요 없습니다.

🔧 (엔지니어용) 왜 방향이 바뀌는지 + 확인 방법

두 도구가 쓰는 Windows 성능 카운터가 다릅니다.

• 모니터링 프로그램: % Processor Time (GetSystemTimes()) — CPU가 바빴던 시간 비율
• 작업 관리자(Win 8+): % Processor Utility — 기본 주파수 기준 실효 일량

두 값의 관계는 대략 주파수 비율(= 현재 주파수 ÷ 기본 주파수)로 결정됩니다.

% Processor Utility ≈ % Processor Time × (현재 주파수 ÷ 기본 주파수)

• 절전으로 기본보다 느리면 비율<1 → Utility < Time → 모니터링 값이 더 높음
• 터보로 기본보다 빠르면 비율>1 → Utility > Time → 작업 관리자가 더 높음
• 예) Time 50%일 때, 절반 속도면 TM 25% / 터보 1.3배면 TM 65%

Windows 7 작업 관리자는 % Processor Time을 그대로 쓰므로 모니터링 값과 거의 일치합니다.

PowerShell로 두 카운터를 동시에 비교:

Get-Counter '\Processor(_Total)\% Processor Time', '\Processor Information(_Total)\% Processor Utility'

상세 기술 설명은 docs/COLLECTORS.md의 "Windows 작업관리자와의 값 차이" 절을 참고하세요.

결론. 모니터링 프로그램과 작업 관리자의 CPU 값이 다른 것은 측정 방식의 차이일 뿐 정상이며, CPU 속도 상태에 따라 어느 쪽이 높을지도 바뀝니다. 두 값 모두 정확하니, 그대로 사용하셔도 됩니다.
